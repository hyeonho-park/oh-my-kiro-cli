# Frontend UI/UX Skill

## Designer-Developer Mode

Enhanced frontend and UI/UX capabilities.

### Design Principles

1. **User-First**: Every decision serves the user
2. **Accessibility**: WCAG 2.1 AA compliance
3. **Performance**: Fast load, smooth interactions
4. **Consistency**: Unified design language
5. **Simplicity**: Remove complexity, not features

### Before Implementing

1. Check design system: existing tokens, components
2. Understand context: current styles, patterns
3. Plan responsive: mobile-first approach
4. Consider a11y: keyboard, screen readers, contrast

### Styling Best Practices

| DO | DON'T |
|----|-------|
| Use design tokens | Hardcode colors/sizes |
| CSS modules/Tailwind | Inline styles everywhere |
| Responsive units (rem, %) | Fixed pixels |
| Semantic HTML | div soup |

### Accessibility Checklist

- Keyboard navigable
- Focus visible
- ARIA labels where needed
- Color contrast 4.5:1+
- Screen reader tested
- No motion for reduced-motion

### Component States

Always design for: Default, Hover, Focus, Active, Disabled, Loading, Error, Empty.
